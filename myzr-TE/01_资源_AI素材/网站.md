## 1. 分类概览

### OpenAI

- 公司 / 产品：ChatGPT、OpenAI API、Codex。
- 通用对话与多模态模型：GPT-3.5、GPT-4、GPT-4o、GPT-5 系列等。
- 推理模型：o 系列，以及后续可能整合到 GPT 系列中的深度推理能力。
- 编程相关：Codex 现在更多指 OpenAI 的编程助手 / 编码代理产品；早期 Codex 是面向代码生成的模型，并曾支撑 GitHub Copilot 的早期能力。

### Anthropic

- 主要产品：Claude 系列。
- 特点：长文本、代码、企业文档处理能力较强。
- 注意：Claude 系列整体为闭源模型，不等同于开源模型。

### Google DeepMind / Google

- 旗舰模型：Gemini Ultra、Gemini Pro 等。
- 高速轻量模型：Gemini Flash。
- 开源 / 开放权重小模型：Gemma 系列。
- 配套产品：Google Gemini 网页端、Google Workspace AI 助手。

## 4. Codex 使用技巧

### Files and folders（文件 / 文件夹）

把本地代码文件或项目目录提供给 Codex 后，Codex 可以读取、分析、修改相关代码。适合代码审查、修复 bug、整理文档、批量重构等任务。

### Goal（目标任务）

给 Codex 设定持续目标，让它围绕同一个任务反复推进，直到完成或遇到明确阻塞。适合较长任务，例如多文件重构、持续排查、分阶段整理资料。

### Plan mode（规划模式）

先进入规划模式，让 Codex 输出执行方案；确认后再开始改文件、运行命令。适合风险较高或范围不清楚的任务。

## 5. 模型选择建议

### 5.6 Sol

- 定位：全能旗舰，推理深度高，消耗 token 多，成本较高。
- 适合：大型项目重构、复杂嵌入式底层问题、Linux 内核 / 驱动分析、复杂 bug 溯源。
- 示例场景：RK3588 / i.MX8MP 驱动、Linux 内核排查、大型代码库分析。

### 5.6 Terra

- 定位：能力与速度较均衡，日常开发性价比较高。
- 适合：写 demo、完善注释、修复单文件 bug、整理 Typora 文档、中等规模代码修改。

### 5.6 Luna

- 定位：轻量快速，响应速度快，成本较低。
- 适合：小段 C 工具函数、命令行脚本、简单语法查询、短文本问答。

### GPT-5.5

- 定位：稳定成熟的旧旗舰。
- 适合：追求稳定、避免版本兼容警告、长期写嵌入式代码或常规开发。
- 备注：如果当前 Codex 版本较旧，使用较新的模型名可能出现 `model metadata not found` 一类警告。

### GPT-5.4 / GPT-5.4 Mini

- GPT-5.4：适合简单脚本和普通问答，复杂代码能力弱于新旗舰。
- GPT-5.4 Mini：速度快、成本低，但不适合多文件工程调试和复杂驱动分析。

### GPT-5.2

- 定位：旧版本。
- 建议：不推荐用于复杂代码理解、工程调试或多文件任务。

## 6. 分场景推荐

| 使用场景                                             | 首选      | 备选                |
| ---------------------------------------------------- | --------- | ------------------- |
| 嵌入式 Linux 驱动、大型项目重构、底层 bug 排查       | 5.6 Sol   | 5.6 Terra / GPT-5.5 |
| 写 demo、完善文档、单文件代码修改、Typora 笔记整理   | 5.6 Terra | 5.6 Luna / GPT-5.5  |
| 临时小段代码、简单语法查询、快速问答                 | 5.6 Luna  | 5.4 Mini            |
| 想消除黄色 `model metadata not found` 警告、追求稳定 | GPT-5.5   | GPT-5.4             |
| 极简单文字问答、极小代码片段、想省 token             | 5.4 Mini  | 5.6 Luna            |
