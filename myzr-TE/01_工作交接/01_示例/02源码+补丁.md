



# 编译步骤

初始源码

```
/home/chensz/my-work/03_aw/01_t536/02_aw/01_source/T536_V1.1.tar.gz
```

解压成`.repo`		解压后，会自动新建一个文件夹v1.1

```
tar xvf
#前提
lizj@myzr-u2004-ec50:~/my-work/RK3588$ ls -F
rk3588_linux_release_v1.6.0_20250413.tgz  V1.1/
lizj@myzr-u2004-ec50:~/my-work/RK3588$ 
#压缩文件放在RK3588里面
#执行命令
tar xvf rk3588_linux_release_v1.6.0_20250413.tgz -C ./V1.1
```

恢复本地

```
./.repo/repo/repo sync -l
```

离线一键全盘复原源码，改坏代码后恢复出厂原始源码状态，全程不用联网

`repo sync -l`：**只动本地、不碰外网**，只用本机已存缓存还原代码，不会升级源码版本。

myzr远程仓库

```
http://git.myzr.lan:3000/linux/t536-linux-5.10.198.git
```

```python
# -b dev 指定分支；-n 只下载.git不落地源码；. 代表克隆到【当前目录】不新建文件夹；--depth=1只拉最新1个提交
git clone -b dev -n --depth=1 --filter=blob:none http://git.myzr.lan:3000/linux/t536-linux-5.10.198.git .

# 开启稀疏检出，只单独拉出根目录.gitignore
git sparse-checkout init --no-cone
echo ".gitignore" >> .git/info/sparse-checkout

# 检出dev分支，只生成.gitignore
git checkout dev
```



拷贝git到==源码==下

```
#chensz
cp ../*/.git* ./ -r
#lizj
# 复制文件
cp -rf t536-linux-5.10.198/* V1.1/
#-r：递归复制目录（.git 是隐藏目录，里面一堆子文件子文件夹，不加 - r 无法拷贝）
cp -r /home/lizj/my-work/RK3588/rk3588-linux-5.10.226/.git .
```





```
#切换代码分支，把源码切到 dev 开发分支
git checkout dev
```



```
#丢弃你在本地所有私自修改、新增的改动文件，全部恢复成 dev 分支原始文件
git restore .
```

`.`代表**当前目录所有文件**，只要你改了驱动 / 内核配置，这条命令一键撤销所有改动，源码回到刚切完 dev 的干净出厂状态。



1. 你之前改乱代码、测试改了一堆配置，**切 dev 分支 + 一键清改动**，源码回到标准开发版；
2. 清理完脏文件，才能干净地从头编译 T536 固件（对应备注：清理编译残留再编译）。
