LXD **是一套完整可用的 Ubuntu 文件系统**（有 /bin、/usr、apt、用户、环境变量，命令、软件用法和真机 Ubuntu 完全一致），用来**统一固定编译环境**，避免不同 Ubuntu / 系统版本、库版本差异导致编译报错；全项目开发人员共用一套容器环境，SDK 编译结果一致性。

轻量级的虚拟电脑，Ubuntu分身

宿主机：真正安装在机器上的 Ubuntu（母体）

LXD 容器：**独立的 Ubuntu 分身**，共用宿主机 Linux 内核，但拥有自己全套系统文件、软件目录。

- 在容器`apt install gcc`，软件只装在容器内部`/usr`，宿主机不会多出这个包；

- 容器误删系统文件崩掉，宿主机系统安然无恙。

> 作用：隔绝宿主机系统干扰，一套容器 = 统一编译环境。

## 2. SDK 依赖部署

Software Development Kit：软件开发工具包

在容器内安装 RK 平台依赖库：交叉编译工具链、buildroot 依赖、gstreamer/v4l2/ISP 驱动依赖、摄像头 / 以太网组件库，补齐源码编译所需系统环境。

在容器里**用 apt 安装主机侧（LXD）编译工具 & 系统依赖**：gcc、make、dtc、各类开发库（buildroot/gstreamer/v4l2 编译依赖）。

## 3. 源码构建

拉取 RK 原厂内核、Buildroot 根文件系统、U-Boot、Sensor 驱动、应用源码，配置内核 DTS、Buildroot 配置项（勾选 gst/v4l2 / 蓝牙等组件）。

## 4. 编译

执行 buildroot make、内核 make、uboot 编译，**交叉编译生成：u-boot.bin、kernel 镜像、rootfs 根文件系统**。

## 5. 镜像打包

用瑞芯微打包工具，把 uboot+kernel+rootfs + 分区参数合并成**完整固件镜像 (.img)**，即开发板可烧录的整机系统。

## 6. 驱动烧录适配

1. 前期：DTS / 驱动源码已经编译进镜像；
2. 烧录：通过 USB/TF 卡将完整镜像写入开发板 eMMC/NVMe 闪存，驱动随系统开机自动加载。

## 7. 烧录工具量产

量产阶段：使用 RK 批量烧录工具，批量给成品板子一键烧录固件，工厂量产使用。

## 8. 功能测试

板子上电后逐项验证：3 路摄像头预览、以太网 eth0/1/2、WiFi / 蓝牙、按键、NVMe 存储等硬件功能（就是你之前`evtest/gst-launch/ifconfig`的实操环节）。

**不用硬件改 DTS、不用驱动裁内核、不用软件裁 Buildroot，最后全编译出新镜像**

- 硬件外设裁剪（不用的硬件禁用：HDMI/USB/ 网口 / SD 卡 / PCIe / 摄像头等）【DTS 修改】路径：kernel/arch/arm64/boot/dts/rockchip/ 你的板级 dts

- 内核功能裁剪（去掉不用驱动：声卡 / 网卡 / 文件系统 / 调试模块）【内核.config】路径：kernel 目录

- 根文件系统裁剪（删应用 / 库 / 服务：桌面、SSH、播放器、多余工具）【Buildroot】路径：buildroot 目录
