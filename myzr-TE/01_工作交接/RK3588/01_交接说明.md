# 编译步骤

1.初始源码路径

```
/home/chensz/my-work/01_rk/01_rk3588/02_myzr/01_rk_source/rk3588_linux_release_v1.6.0_20250413.tgz
```



2.解压成`.repo`

```
tar xvf 
```

3.恢复本地

```
./.repo/repo/repo sync -l
```

`-l = --local-only`：**只使用本地已缓存代码，不拉取远端服务器源码**

`sync`：同步拉代码

联网从服务器下载全套 RK3588 源码，自动补齐 kernel、uboot、驱动等所有文件夹，让本地代码和服务器保持一模一样。



**离线恢复源码，不需要联网下载**

你这个 tgz 压缩包提前打包好了完整`.repo`仓库索引 + 所有源码缓存，`-l`参数**跳过访问外网 git 服务器**，只用`.repo`里本地缓存文件，在当前目录还原出全部工程源码（kernel、u-boot、buildroot、app 等文件夹）。

普通`repo sync`（不带 - l）：会联网去远端拉代码，没网 / 内网隔离环境直接失败；

`repo sync -l`：纯本地解压还原源码，**断网也能完整拉出工程目录**，正是瑞芯原厂预打包源码包的用法。

执行完当前`3588`目录就从只有`.repo`隐藏文件夹 → 生成完整整套 RK3588 源码目录。

4.myzr远程仓库

```
http://git.myzr.lan:3000/linux/rk3588-linux-5.10.226.git
```

5.拷贝git到源码下

6.切换到dev

```
git checkout dev
```

```
git restore .
```
