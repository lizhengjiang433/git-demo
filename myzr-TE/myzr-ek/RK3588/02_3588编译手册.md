## 编译环境要求

编译主机需在ubuntu系统中进行，笔者主机系统为Ubuntu 20.04

注意：如果有代理需要把代理关掉

## 下载源码包

1.下载rk3588源码包，网盘路径路径为：3.软件资料-->3.1源码-->rk3588-linux-5.10.226.tar.gz.part-a*

2.创建编译目录:

```Bash
mkdir -p ~/my-work/RK3588/02_sources/
```

3.把源码放到刚创建的目录中，并进行合并：

```
cat rk3588-linux-5.10.226.tar.gz.part-* > rk3588-linux-5.10.226.tar.gz
```

4.解压出`rk3588-linux-5.10.226`源码目录

```Bash
tar xvf rk3588-linux.tar.xz -C ~/my-work/RK3588/02_sources/
```

## 编译步骤

### 编译配置选择

在`rk3588-linux-5.10.226`源码顶层目录下

```
./build.sh chip
```

输入`3`选择`myzr_rk3588_ek360_defconfig`

```
1. rockchip_defconfig
2. myzr_rk3588_ek360_amp_defconfig
3. myzr_rk3588_ek360_defconfig
4. myzr_rk3588_ek360_mcu_defconfig
5. rockchip_rk3588_evb1_lp4_v10_amp_defconfig
6. rockchip_rk3588_evb1_lp4_v10_defconfig
7. rockchip_rk3588_evb1_lp4_v10_mcu_defconfig
8. rockchip_rk3588_evb7_v11_defconfig
9. rockchip_rk3588_ipc_evb1_v10_defconfig
10. rockchip_rk3588_ipc_evb7_lp4_v11_defconfig
11. rockchip_rk3588_multi_ipc_evb1_v10_defconfig
12. rockchip_rk3588s_evb1_lp4x_v10_defconfig
Which would you like? [1]: 3
```

### 整体编译

1.运行整体编译(编译时间较长)，输入如下命令:

**默认rootfs是buildroot**

```Bash
./build.sh 
```

2.编译成功后在rockdev/目录下可看到相关镜像，其中update.img是所有镜像的集合

### 单独编译uboot

```Bash
./build.sh uboot
```

### 单独编译Kernel

```Bash
./build.sh kernel
```

### 单独编译rootfs

在SDK主目录下输入如下命令:

**buildroot**

```Bash
./build.sh buildroot
```

**debian12**

```Plain
./build.sh debian
```

### 打包固件

```Bash
./build.sh firmware
```

### 打包update.img

```Bash
./build.sh updateimg
```

### 更多编译命令

```Plain
./build.sh help
```

完成上述操作后可按照刷机手册重新刷机
