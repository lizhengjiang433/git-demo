### RS485

接口丝印：J20

测试说明：通过485-USB转换头与电脑互联进行收发测试

测试操作

1.使用485-USB转换头连接开发板和电脑

2.使用`Xshell`打开对应串口,将波特兰设置成115200，数据为8位，停止位1位

3.发送测试，输入如下命令

```Bash
num=1
stty -F /dev/ttyAS$num 115200 -crtscts
#该引脚接 RS485 模块的DE+RE收发控制脚。export为普通可写字符设备文件
#把引脚设置为输出通道
echo 200 > /sys/class/gpio/export
# 设置为输出
echo out > /sys/class/gpio/gpio200/direction
#GPIO 输出高电平，485 进入发送模式，总线允许向外发数据。0则接收使能
echo 1 > /sys/class/gpio/gpio200/value
echo MYZR > /dev/ttyAS$num
```

可以看到485串口终端输出`MYZR`

-F指定串口设备

`-crtscts`：关闭硬件流控（RTS/CTS），485 不需要串口硬件流控，避免冲突

DE = 驱动使能，RE = 接收使能；485 半双工逻辑：引脚输出高 = 发送模式，低 = 接收模式。

全志芯片 GPIO 编号计算公式：

```
GPIO编号 = 端口组偏移 + 引脚序号
```

例：

- PA0 → 0

- PB0 → 32

- PC0 → 64

- PD0 → 96

- PE0 → 128

- PF0 → 160

- PG0 → 192

  PG8 = 192 + 8 = 200

  开发板硬件原理图里，RS485 的 DE/RE 控制引脚固定分配为 PG8，内核全局编号就是 200，所以必须写 200，不能换其他数字。

4.接收测试，在485串口终端，直接输入123再按回车键发送，可以看到信息

```Bash
echo 200 > /sys/class/gpio/export
echo out > /sys/class/gpio/gpio200/direction
echo 0 > /sys/class/gpio/gpio200/value
cat /dev/ttyAS$num
...
123
```

按下"Ctrl + C"退出，操作过程中的输出符合预期即功能正常。

其他串口gpio，uart2（118）、uart3（230）、uart4（271）、uart9（261）、uart11（238）
