```
#将这个空文件转换为ext4文件系统
mkfs.ext4 -F -E nodiscard -O ^metadata_csum,^64bit sdcard_rootfs.ext4
```

mkfs= make filesystem

-F:force,强制格式化

`-E` = extended options 扩展属性选项，后面逗号分隔多个扩展配置

iscard：内核支持**在线丢弃空闲块**（TRIM，固态硬盘 / 闪存优化，自动回收空白块）

`nodiscard`：关闭 discard 功能，不开启空闲块自动回收

`-O` = feature，开启 / 关闭 ext4 文件系统特性；

`^` 符号代表**禁用该特性**，不带`^`是开启。

## 参数 `-O ^metadata_csum,^64bit`

`-O` = feature，开启 / 关闭 ext4 文件系统特性；

`^` 符号代表**禁用该特性**，不带`^`是开启。

### ① `^metadata_csum` 关闭元数据校验和

- metadata_csum：ext4 新特性，给文件系统元数据（inode、目录、块表）增加 CRC 校验，防止磁盘损坏；

- 加^即关闭：

  老版本 Linux 内核（3.18 及更早 i.MX/RK 嵌入式内核）

  不支持元数据校验，带此特性的 ext4 挂载会直接失败，制作兼容老板子 rootfs 必须关掉。

64bit 关闭 64 位块特性
64bit：ext4 64 位寻址，支持超过 16TB 超大存储；
加^即关闭，使用传统 32 位寻址：
嵌入式 SD 卡、rootfs 镜像容量普遍几百 MB~ 几 GB，完全用不到 64 位寻址；老内核同样不兼容 64bit ext4，关闭保证全兼容。
