# 应用项目



电子产品量产工具

通过`gui.conf`配置文件定义，新增硬件模块只需新增测试脚本，不用修改 GUI 主程序；各测试程序解耦互不干扰。

整套工具无复杂桌面系统，基于 Linux 小型根文件系统，GUI 基于 tslib 触摸屏库 + FreeType 矢量字体，资源占用极低，适配 IMX6ULL 这类低端 ARM 主控。

tools文件解读：

- genimage镜像分区打包工具

- sdcard_genimage.cfg：SD卡镜像打包配置文件，专门用于生成工装SD卡镜像

- emmc_genimage.cfg：eMMC 成品系统打包配置文件，于生成最终产品固件 emmc.img

  打包工装 SD 镜像时，会把emmc.img放进 sdcard.img 的文件系统中；
  产线上单板从 SD 卡启动工装系统后，自动读取内部的emmc.img，dd 写入板载 eMMC 闪存；
  测试完成、拔掉 SD 卡，切换拨码从 eMMC 启动，设备运行的就是这套干净成品系统，完全不存在产线测试相关程序。



触摸屏选择：IMX6ULL LCD 接口是 3.3V，

项目 GUI 基于 `/dev/fb0` Framebuffer，IMX6ULL 开发板硬件只引出 RGBLCD 排线

### 电阻触摸：

**靠压力按压让两层薄膜接触导通**，通过电压分压计算触摸坐标。

仅单点

### 电容触摸：

屏幕内置感应电极，依靠**人体手指导电产生微弱电容变化**定位；只有导电人体 / 专用电容笔才能触发，纯压力无效。

容易漂移，需要定期 tslib 校准

制作镜像

u-boot-dtb.imx 通常指的是 NXP i.MX 平台用的启动镜像/二进制镜像，启动加载镜像，初始化最基础硬件，等待加载设备树和内核

| 后缀       | 含义                | 流程                  |
| ---------- | ------------------- | --------------------- |
| `.tar`     | 仅 tar 打包，无压缩 | 只打包，不压缩        |
| `.tar.gz`  | gzip 压缩的 tar 包  | tar 打包 → gzip 压缩  |
| `.tar.bz2` | bzip2 压缩的 tar 包 | tar 打包 → bzip2 压缩 |
| `.tar.xz`  | xz 高压缩 tar 包    | tar 打包 → xz 压缩    |

bzip2 是一款 Linux 经典无损压缩算法，压缩率比 gzip 更高。

它**只能压缩单个文件**，不能直接压缩一堆文件夹

tar 能把一堆文件 / 目录打包成**单个.tar 文件**，但不会压缩；

gzip开发板自带

## 准备配置文件

已经生成好的 `u-boot-dtb.imx` 打包进 `emmc.img`

emmc_genimage.cfg

```
image emmc.img {
hdimage {
}
partition u-boot {
in-partition-table = "no"
image = "u-boot-dtb.imx"
offset = 1024
}
partition arduino {
partition-type = 0xC
size = 50M
offset = 10M
}
partition rootfs {
partition-type = 0x83
image = "emmc_rootfs.ext4"
size = 2048M
}
}
```

其中u-boot 分区不在分区表里，即你只可以看到后面2 个分区。u-boot 分区从偏移地址1024 开始

sdcard_genimage.cfg

rootfs 分区有2048M，里面的内容来自emmc_rootfs.ext4。

## 制作镜像文件

原料sdcard_rootfs.ext+u-boot-dtb.imx

sdcard.img

```
mkdir tmp_sdcard_rootfs
#将文件解压到tmp_sdcard_rootfs目录下
sudo tar xjf sdcard_rootfs.tar.bz2 -C tmp_sdcard_rootfs/
```

```
#创建名为 sdcard_rootfs.ext4 的空文件
dd if=/dev/zero of=sdcard_rootfs.ext4 bs=1M count=650
#将这个空文件转换为ext4文件系统
mkfs.ext4 -F -E nodiscard -O ^metadata_csum,^64bit sdcard_rootfs.ext4

mkdir tmp
#挂载sdcard_rootfs.ext4 镜像到tmp目录
sudo mount -t ext4 sdcard_rootfs.ext4 tmp
#复制tmp_sdcard_rootfs文件里面的内容到tmp
sudo cp -rfd tmp_sdcard_rootfs/* tmp/
sudo umount tmp
```

挂载，把镜像文件临时打开为一个可以访问的目录

`tmp` 此时相当于这份镜像里的根目录入口

没挂载时：它像一个“压缩包”

挂载后：它像“已经打开的压缩包文件夹”

先做一个空的 `ext4` 镜像文件

再把 `tmp_sdcard_rootfs` 里的系统文件复制进去

最后得到可供后续打包的 `sdcard_rootfs.ext4`
