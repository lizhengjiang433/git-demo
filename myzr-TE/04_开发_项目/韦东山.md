# 应用项目



电子产品量产工具

通过`gui.conf`配置文件定义，新增硬件模块只需新增测试脚本，不用修改 GUI 主程序；各测试程序解耦互不干扰。

整套工具无复杂桌面系统，基于 Linux 小型根文件系统，GUI 基于 tslib 触摸屏库 + FreeType 矢量字体，资源占用极低，适配 IMX6ULL 这类低端 ARM 主控。

tools文件解读：

- genimage镜像分区打包工具

- sdcard_genimage.cfg：SD卡镜像打包配置文件，专门用于生成工装SD卡镜像

- emmc_genimage.cfg：eMMC 成品系统打包配置文件，于生成最终产品固件 emmc.img

  打包工装 SD 镜像时，会把emmc.img放进 sdcard.img 的文件系统中；
  产线上单板从 SD 卡启动工装系统后，自动读取内部的emmc.img，dd 写入板载 eMMC 闪存；
  测试完成、拔掉 SD 卡，切换拨码从 eMMC 启动，设备运行的就是这套干净成品系统，完全不存在产线测试相关程序。



触摸屏选择：IMX6ULL LCD 接口是 3.3V，

项目 GUI 基于 `/dev/fb0` Framebuffer，IMX6ULL 开发板硬件只引出 RGBLCD 排线

### 电阻触摸：

**靠压力按压让两层薄膜接触导通**，通过电压分压计算触摸坐标。

仅单点

### 电容触摸：

屏幕内置感应电极，依靠**人体手指导电产生微弱电容变化**定位；只有导电人体 / 专用电容笔才能触发，纯压力无效。

容易漂移，需要定期 tslib 校准

支持多点
